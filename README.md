# TCP 文件传输系统

基于 Python 3 的 TCP 文件传输实验项目，包含多线程并发服务器、自定义应用层协议、分块缓存读写、**浏览器 Web 控制台**，以及 CustomTkinter 桌面客户端界面。

## 运行环境要求

- **Python**：3.8 及以上
- **依赖库**：
  - `customtkinter`（桌面客户端 GUI，Web 访问无需安装）
  - Python 标准库：`socket`、`threading`、`struct`、`json`、`os`、`http.server`

安装依赖（仅桌面客户端需要）：

```bash
pip install customtkinter
```

## 自定义应用层协议（防粘包）

每帧报文由 **固定 10 字节报文头** + **变长 Payload** 组成：

| 字段 | 长度 | 说明 |
|------|------|------|
| Magic | 4 字节 | 固定 `FTCP`，识别合法帧 |
| Version | 1 字节 | 协议版本 `0x01` |
| Command | 1 字节 | 操作类型（登录/列表/上传/下载/上传ACK 等） |
| PayloadLen | 4 字节 | 大端 uint32，Payload 字节数 |

**防粘包思路**：接收端先用 `recv_exact(10)` 读满固定头，再根据 `PayloadLen` 精确读取 Payload；文件体按 `file_size` 循环 `recv(4096)`，发送端对应循环 `read(4096)` + `sendall()`，保证边界清晰。

**文件传输元数据**（Payload 内）：`filename_len(4)` + `file_size(8)` + `filename(UTF-8)`，随后紧跟原始二进制文件流；上传结束后服务器返回 **`CMD_UPLOAD_ACK`** 确认帧（含保存字节数与文件名），客户端收到 ACK 后才判定上传成功。

## 公网访问说明

本系统服务端监听 **`0.0.0.0:8082`**，同一端口同时支持两种访问方式，连接建立后由服务器自动识别协议类型：

| 访问方式 | 地址示例 | 协议 | 适用场景 |
|----------|----------|------|----------|
| **浏览器 Web 控制台** | `http://8.134.97.118:8082/` | HTTP | 任意设备，免安装，打开即用 |
| **Python 桌面客户端** | IP `8.134.97.118`，端口 `8082` | FTCP（自定义 TCP） | 实验报告、进度条、本地 GUI 体验 |

> 将 `8.134.97.118` 替换为你云服务器的**公网 IP** 即可。

### 公网部署步骤

**1. 在云服务器上启动服务**

```bash
cd server
python3 server.py
```

启动成功后终端会显示：

```
TCP 文件传输服务器已启动 -> 0.0.0.0:8082
Web 控制台访问 -> http://<公网IP>:8082/
文件存储目录: .../server/storage
```

**2. 放行 8082 端口（必做）**

| 配置项 | 说明 |
|--------|------|
| 云厂商安全组 | 入方向添加规则：协议 **TCP**，端口 **8082**，来源 `0.0.0.0/0`（或指定 IP） |
| 系统防火墙 | 若启用 `firewalld`/`ufw`，需额外放行 8082 |

```bash
# firewalld 示例
sudo firewall-cmd --add-port=8082/tcp --permanent && sudo firewall-cmd --reload

# ufw 示例
sudo ufw allow 8082/tcp
```

**3. 验证公网是否可达**

- 浏览器访问：`http://你的公网IP:8082/`
- 或在本地执行：`curl -I http://你的公网IP:8082/`  
  正常应返回 `HTTP/1.0 200 OK` 或 `HTTP/1.1 200 OK`

### Web 控制台操作（浏览器）

### 桌面客户端操作

#### 连接服务器

1. 在 **服务器 IP** 输入框填写服务器地址（本机测试填 `127.0.0.1`，公网填 `8.134.97.118` 等）。

1. 输入用户名和密码（如 `admin` / `admin123`）。

3. 成功则状态变为「已登录」；失败会弹窗提示具体原因。

3. 点击 **「下载文件」**。
4. 下载完成后文件保存在 `client/downloads/`。

## 技术要点说明

- **协议分流**：8082 端口根据连接首包自动区分 HTTP（`GET`/`POST`…）与 FTCP（魔数 `FTCP`），Web 与 TCP 客户端互不干扰
- **多线程并发**：服务器 `accept()` 后为每个客户端创建 `threading.Thread` 工作线程
- **分块缓存**：TCP 传输读写均使用 **4096 字节** Buffer，符合流式传输实验要求
- **上传 ACK**：FTCP 上传完成后服务器返回 `CMD_UPLOAD_ACK`，客户端确认后才判定成功
- **GUI 不阻塞**：桌面客户端 Socket 操作在子线程执行，进度通过回调更新主线程 UI

## 常见问题

| 现象 | 可能原因 | 处理 |
|------|----------|------|
| 浏览器显示「无法访问此网站」 | 服务未启动或 8082 未放行 | 确认 `python3 server.py` 运行中，检查安全组/防火墙 |
| 页面能打开但登录失败 | 账号密码错误 | 对照 `users.json`，注意区分「用户名错误」与「密码错误」 |
| 桌面客户端连接失败 | IP/端口错误或端口未开放 | 公网 IP 填云服务器地址，端口固定 `8082` |
| 下载失败「文件不存在」 | storage 中无该文件 | 先上传，或检查 `server/storage/` |
| Web 上传成功但列表为空 | 未登录或未点刷新 | 登录后点击「刷新列表」 |
| 缺少 customtkinter | 未安装桌面客户端依赖 | 执行 `pip install customtkinter`（Web 访问不需要） |
